# THE BODY PROTECTION COMPANY
## DESIGN SYSTEM
### Version 1.0

This document translates the Brand Bible into a practical visual and creative execution system. It is intended for designers, marketers, developers, content creators, and external partners building assets for The Body Protection Company.

---

# 1. PURPOSE

The Body Protection Company is not a supplement brand. It is a precision-driven biological systems brand. The visual system must communicate clinical rigor, security, and composure. Every asset should feel engineered, not decorated.

This design system exists to ensure that all output is coherent, recognizable, and aligned with the company’s strategic foundation.

# 3. CREATIVE PRINCIPLES

## 3.1 Precision Over Expression
Design should feel exact, measured, and deliberate. Avoid flourish, decoration, or unnecessary stylistic noise.

## 3.2 Clinical, Not Wellness
Do not drift into soft wellness, spa, natural health, or lifestyle supplement aesthetics.

## 3.3 Authority Through Restraint
The brand should feel confident enough not to over-explain. Use negative space, strong hierarchy, and minimal copy.

## 3.4 System Over Product
The company should feel like a modular optimization platform, not a collection of random SKUs.

## 3.5 Hard Contrast Wins
High contrast, clear geometry, and defined structure are always preferred over subtlety.

# 5. COLOR SYSTEM

The palette should feel sterile, technical, and forceful. Use color sparingly and intentionally.

## Primary Colors
- Void Black: #000000
- Clinical White: #F3F1ED
- Signal Orange: #F58220

## Supporting Neutrals
- Hard White: #FFFFFF
- Divider Gray: #B8B8B8
- Structural Gray: #D9D9D9
- Light Panel Gray: #ECEAE6

## Color Usage Rules
- Black is used for authority, containers, contrast, and visual mass.
- White and off-white are used for clarity, breathing room, and premium restraint.
- Orange is used for emphasis, titles, highlights, and system identity.
- Gray is used for lines, separators, panels, and quiet structural support.

## Color Ratio Guidance
Typical layout ratio:
- 50 to 70 percent white / off-white
- 20 to 35 percent black
- 5 to 15 percent orange

## Do Not
- No gradients
- No glows
- No soft shadows
- No rainbow accent systems
- No neon biotech cliché treatments
- No overuse of orange

# 6. TYPOGRAPHY SYSTEM

Typography should feel sharp, technical, and modern. The hierarchy is bold and compressed in feel, even if the font itself is not condensed.

## Primary Typeface
Inter

Recommended weights:
- Bold
- SemiBold
- Regular
- Light

Use for:
- Headlines
- Section titles
- Subheads
- Body copy
- Buttons
- Navigation

## Secondary Typeface
IBM Plex Mono

Recommended weights:
- Medium
- Regular

Use only for:
- Batch numbers
- Technical callouts
- Labels
- Data points
- Protocol codes
- Verification references
- Product identifiers
- Footer metadata
- Scientific overlays

## Typography Tone
- Headlines should feel assertive and controlled
- Body should be sparse and readable
- Mono text should feel technical, never decorative

---

# 7. TYPE HIERARCHY

## H1
Use for primary slide or page titles.

- Font: Inter Bold
- Case: Uppercase preferred
- Size: 64 to 96 px for slides
- Line height: 0.9 to 1.0
- Tracking: tight
- Usage: large, dominant, aggressive

Examples:
- BODY PROTECTION COMPANY.
- SYSTEM PHILOSOPHY
- METABOLIC MODULE

## H2
Use for section headers and major subheads.

- Font: Inter Bold
- Size: 36 to 56 px
- Line height: 0.95 to 1.05
- Case: uppercase or title case depending on context

## H3
Use for supporting section titles.

- Font: Inter SemiBold or Bold
- Size: 20 to 28 px
- Line height: 1.1 to 1.2

## Body
- Font: Inter Regular or Light
- Size: 12 to 16 px
- Line height: 1.35 to 1.55
- Keep paragraphs short
- Avoid dense blocks

## Data / Label / Caption
- Font: IBM Plex Mono
- Size: 10 to 13 px
- Tracking: normal or slightly open
- Case: uppercase preferred

# 8. GRID AND SPACING

## Grid System
Use a strict grid. Layouts should feel mathematical and architectural.

Recommended:
- 12-column grid for digital layouts
- Modular rectangular divisions for slides
- Strong vertical and horizontal alignment
- Repetition of hard rectangular zones

## Spacing Scale
Use an 8-point system:
- 8
- 16
- 24
- 32
- 48
- 64
- 96
- 128

## Spacing Rules
- Use generous negative space
- Never cram content
- Leave breathing room around headlines
- Large empty areas are a feature, not a problem

---

# 9. LAYOUT LANGUAGE

## General Layout Behavior
The brand uses structured, hard-edged, asymmetrical compositions. Layouts should feel like system diagrams or architectural elevations.

## Recurring Layout Patterns

### Pattern A: Split Screen
One side contains information. The other side contains a black or neutral visual mass block.

### Pattern B: Modular Card Grid
Used for systems, module overviews, capability grids, and future roadmap sections.

### Pattern C: Dominant Headline + Quiet Supporting Copy
Large aggressive title, with smaller supporting explanation positioned cleanly below or beside it.

### Pattern D: Title + Block + Data
A strong section title, a large structural shape, and a small data or proof callout.

## Composition Rules
- Favor asymmetry over centered balance
- Use hard vertical cuts
- Use black blocks as structural anchors
- Avoid floating elements without alignment
- Do not fill every empty area

---

# 10. SIGNATURE BRAND DEVICES

## 10.1 Black Structural Blocks
These are a core part of the brand language.

Purpose:
- Provide visual tension
- Signal modularity
- Add weight and authority
- Create contrast against typographic areas

Rules:
- Use pure black
- Rectangles only
- No gradients, textures, or overlays unless explicitly art directed
- Can be used as image placeholders, dead space, or framing elements

## 10.2 Thin Divider Lines
Use subtle but visible linework to reinforce structure.

Rules:
- 1px lines preferred
- Gray or black depending on contrast
- Use to divide sections, modules, and footers
- Avoid decorative linework

## 10.3 Utility Footer Tags
Small technical labels or callout bars can be used in IBM Plex Mono to add system credibility.

Examples:
- Optimized for metabolic regulation.
- Recovery becomes a repeatable advantage.
- Each peptide serves a specific biological function.

Use icons sparingly.

# 14. COMPONENT RULES

## Buttons

### Primary Button
- Background: black
- Text: white

- Case: title case or uppercase depending on environment
- Shape: square or slightly softened only if product requires it

### Secondary Button

- Minimal styling
- Strong padding

## Tags / Labels
- IBM Plex Mono
- Small size
- Highly controlled use

# 15. MODULE SYSTEM

The brand should present products as modules within a broader optimization system.

## Module Naming Pattern
Use the structure:
- METABOLIC MODULE
- RECOVERY MODULE
- HORMONAL MODULE
- AESTHETIC MODULE
- CELLULAR PERFORMANCE
- FAT LOSS MODULE

## Module Page Structure
A typical module layout should include:
1. Module title
2. Compound or stack name
3. Short functional descriptor
4. Short explanation of mechanism or benefit
5. Technical footer or proof tag
6. Large structural block or image area

## Module Copy Style
- Short
- Direct
- Clinical
- Functional
- No hype

Example:
Advanced metabolic regulation through targeted receptor activity.

Not:
Torch fat fast and transform your body.

# 20. DO / DO NOT

## Do

- Use negative space generously
- Use black blocks as signature structure

- Maintain a clinical, premium tone
- Use IBM Plex Mono for data and technical emphasis

## Do Not
- Use gradients
- Use glows

- Use fitness clichés
- Use supplement clichés
- Use soft wellness visuals
- Use bro-science language
- Use decorative design tricks to create fake sophistication
- Overwrite copy
- Overuse orange

---

# 21. APPROVAL FILTER

Before approving any asset, ask:

1. Does this feel clinical, secure, and precise?
2. Does this look like a system rather than a supplement ad?
3. Does this avoid wellness softness and fitness cliché?
4. Does the typography feel structured and authoritative?
5. Is the message direct and mechanism-oriented?
6. Would this feel at home in a premium biotech environment?
7. Is there any hype, noise, or decorative excess that should be removed?

If the answer to any of these is no, the asset is not ready.

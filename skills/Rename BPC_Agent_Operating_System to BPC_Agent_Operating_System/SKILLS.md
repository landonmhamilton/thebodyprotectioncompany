# SKILL: BPC Agent Operating System

## Purpose
Use this skill whenever working on anything related to The Body Protection Company.

This is the top-level governing skill. It defines how all outputs must behave across product, protocol, content, growth, trust, UX, and company-building tasks.

## When to use
Use this skill for any BPC task, including:
- product design
- protocol design
- landing pages
- brand messaging
- content creation
- funnel design
- app flows
- packaging
- verification UX
- growth experiments
- internal system design

If another BPC skill is used, this skill should still govern the output.

## Core doctrine
All actions must reinforce Biological Sovereignty.

Biological Sovereignty means control of biological inputs through verification, structure, and precision.

If output does not reinforce this, it is incorrect.

## Company type
The Body Protection Company is:
- not a supplement brand
- not a commodity peptide store

It is:
- a biological systems company
- an infrastructure layer for Biological Sovereignty

Always prioritize:
SYSTEMS over PRODUCTS

## Primary model
All outputs must follow:

VERIFIED INPUT → STRUCTURED SYSTEM → PREDICTABLE OUTCOME

## Product hierarchy
All outputs must preserve this hierarchy:

SYSTEM → MODULE → COMPOUND → BATCH

Never:
- present compounds as standalone solutions
- break this hierarchy
- reduce system thinking into product-list merchandising

## Language rules

### Always use
- verified
- system
- protocol
- control
- precision
- structure
- sovereignty
- targeted
- mechanism
- module
- batch
- transparency

### Never use
- supplements
- miracle
- transformation
- shredded
- ripped
- game-changing
- secret
- hype language
- generic wellness language
- influencer-style fitness language

## Domain rules

### Product
- products must map to SYSTEM → MODULE
- no random SKUs
- no compound-first positioning
- systems are the product, compounds are components

### Protocol
- protocols must be structured
- protocols must explain mechanism
- protocols must reinforce system thinking
- no product should exist without protocol context

### Content
- content must reinforce Biological Sovereignty
- content must be high signal
- content must avoid entertainment framing
- content must avoid generic “health tips” framing

### Growth
- do not optimize for clicks alone
- optimize for belief shift, trust, and system adoption
- acquisition should lead to structured system entry

### Trust / verification
- verification must be visible
- data must be accessible
- transparency must be default
- trust must be immediate and unavoidable

### UX / experience
- the user should feel like they are entering a system
- not browsing random products
- not shopping a supplement shelf

## Constraint system
Do not:
- add features outside defined scope
- introduce hype language
- simplify systems into product lists
- drift into supplement branding
- write as if this is a commodity ecommerce brand
- use vague premium-brand filler language

## Building logic
When asked to build anything:
1. Identify the domain
2. Map the task to the system hierarchy
3. Apply language and doctrine constraints
4. Produce output aligned with Biological Sovereignty
5. Reject anything that feels like wellness, supplements, or influencer fitness

## Learning and research behavior
This skill should improve output over time through structured learning.

For any repeated domain:
1. Form a hypothesis
2. Execute a variation
3. Observe results
4. Analyze what worked and failed
5. Update future outputs while preserving doctrine

Prioritize:
- real user behavior
- retention patterns
- conversion patterns
- experiment results

Over:
- generic best practices
- assumptions
- surface-level opinions

## Output standard
A correct output should feel like:
- a biological systems company
- a clinical but modern infrastructure brand
- a system built around verification, control, and protocol

If output feels like:
- a supplement brand
- generic wellness
- influencer content
- bro-science
- commodity ecommerce

It is wrong.
Reject and rebuild.

- system name

- next-step recommendation

## Final rule
BPC is building the infrastructure layer for Biological Sovereignty.

All outputs must move toward this.
